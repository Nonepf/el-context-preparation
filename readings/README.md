## 学习方法

### 每天的学习流程

1. **阅读**（1.5–2 小时）：仔细阅读当天的 md 文件，理解每段代码的逻辑
2. **默写**（0.5 小时）：不看书，尝试默写当天的核心代码模板
3. **练习**（1 小时）：做配套练习题，每道题限时
4. **总结**（0.5 小时）：回顾今天卡住的地方，写下笔记

### 比赛时的工作流

1. 打开题目 → 读题 → 纸上画出数据结构和算法流程
2. 复制模板文件 → 在 solve() 函数中写解法
3. 本地测试样例 → 确认输出格式 → 提交
4. 如果 WA（Wrong Answer）：检查边界条件、数据类型、输出格式
5. 如果 TLE（Time Limit Exceeded）：检查算法复杂度是否匹配数据范围

### 复杂度与数据范围对应关系

| 数据范围 n | 可行的最坏复杂度 | 常见算法 |
|-----------|-----------------|----------|
| n ≤ 10 | O(n!) | 全排列枚举 |
| n ≤ 20 | O(2^n) | 状态压缩、子集枚举 |
| n ≤ 100 | O(n³) | Floyd、区间 DP |
| n ≤ 1000 | O(n²) | 简单 DP、Bellman-Ford |
| n ≤ 10^5 | O(n log n) | 排序、Dijkstra 堆优化、二分答案 |
| n ≤ 10^6 | O(n) | 前缀和、滑动窗口、线性筛 |
| n ≤ 10^9 | O(log n) 或 O(1) | 数学公式、二分答案 |

## 核心算法检查表

赛前逐项确认，每项都能在 5 分钟内写出正确代码：

- [ ] 多组数据输入输出（C++: `while(t--)`, Python: `for _ in range(t)`）
- [ ] vector 基本操作 + sort 自定义排序
- [ ] unordered_map / unordered_set 计数字段
- [ ] priority_queue（C++ 的小顶堆写法）
- [ ] 一维前缀和 + 差分
- [ ] 二分答案（check 函数 + while(l < r) 模板）
- [ ] 活动选择型贪心 + 反悔贪心
- [ ] 双指针（两数之和） + 滑动窗口（最长无重复子串）
- [ ] 01 背包（一维倒序） + 完全背包（一维正序）
- [ ] 最大子数组和（Kadane）+ LIS（O(n log n)）
- [ ] BFS（网格迷宫） + DFS（连通块）
- [ ] Dijkstra 堆优化模板
- [ ] 拓扑排序（Kahn）
- [ ] 并查集（路径压缩）
- [ ] 字符串哈希（单哈希就够，双哈希更稳）
- [ ] 单调栈（下一个更大元素）
